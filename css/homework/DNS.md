# 도메인과 DNS
## 도메인
이전 서버와 IP에 대하여 알아보았을 때 IP주소를 입력하는 것으로 서버에 접속할 수가 있었다. 그러나 아무런 특징없는 12자리 숫자와 서버나 사이트 이름을 매칭할 수 있는 사람은 많지 않을 것이다. 이를 해결하기 위해 특정 IP에 이름을 붙여 놓은 것이 도메인 또는 도메인 네임이다.

## DNS, Domain Name System
IP에 이름을 붙인 것은 좋으나 이를 컴퓨터가 알아먹지 못하면 아무런 쓸모가 없다. DNS는 도메인 네임을 IP주소로 변환해주는 시스템으로 주소창에 도메인 네임을 입력하면 이를 IP주소로 변환해 실제 서버로 연결시켜주는 역할을 가진다. 또한 이런 DNS를 운용하는 서버를 DNS서버라고 한다.

## DNS서버가 고장나면?
DNS서버가 고장나면(물론 보조 DNS가 있지만 모조리 고장난다면) 도메인 네임과 IP주소를 이어주는 DNS를 이용할 수 없게 되어 원래대로 직접 IP주소를 입력하여 서버에 접속해야하는 사태가 벌어진다. 심할경우 서버가 IP주소로 직접 접속하지 못하게 막았을 경우엔 서버에 접속할 방법이 사라지게 된다. 이를 방지하기 위해 큰 서버들은 자체 DNS서버도 운용하는 경우가 있다.
